Email Validation API

I selected a email validation API from Abstract APIs. The API provides validation for email addresses. It is also able to provide autocorrect for common typos and common domains. I suspect it is simply a parsing tool as through my own testing, any address @gmail.com will show to be valid, and any .com/.org/.edu/etc. domain will show to be valid as well. Therefore it is not able to eliminate all bad email addresses, but is still able to catch a good amount of errors.


One instance where it would be helpful would be if someone were entering their address to recieve more information. If they user typed their email wrong and didn't notice, they would be confused and frustrated as to why they didn't recieve an email.


Another example of where this would be useful is if the user bought something online, and entered an invalid email. If there was something wrong with the order and the seller needed to contact the buyer, it would be bad if the seller could not reach the buyer.


This API is available on https://app.abstractapi.com/. I found it through the provided public API listings on github. The API is fetched using a GET request and providing an email through the query in the URL.
